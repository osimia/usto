# USTO Product Specification

## Цель

USTO - мобильный маркетплейс услуг для Таджикистана, где заказчики публикуют заявки, а мастера находят работу, отправляют отклики, общаются в чате и получают рейтинг.

Основной клиентский продукт: Flutter-приложение для Android и iOS.

## Роли

### Заказчик

Заказчик создает заявки, выбирает мастеров, общается в чате, оплачивает или подтверждает работу и оставляет отзыв.

Основные возможности:

- регистрация и вход по номеру телефона;
- выбор города и района;
- просмотр категорий услуг;
- создание заявки;
- добавление описания, бюджета, срока, адреса и фото;
- просмотр откликов мастеров;
- открытие профиля мастера;
- выбор мастера;
- чат с мастером;
- управление статусами заказа;
- история заказов;
- отзыв и рейтинг после завершения.

### Мастер

Мастер создает профиль, проходит верификацию, смотрит заявки, отправляет отклики и получает заказы.

Основные возможности:

- регистрация и вход по номеру телефона;
- заполнение профиля;
- выбор категорий услуг;
- добавление опыта, описания, портфолио и документов;
- прохождение верификации;
- просмотр ленты заявок;
- фильтры по категории, району, бюджету и срочности;
- отправка отклика с ценой и комментарием;
- чат с заказчиком;
- кошелек и история транзакций;
- история выполненных работ;
- рейтинг и отзывы.

### Администратор

Администратор управляет безопасностью, качеством и контентом платформы.

Основные возможности:

- вход в admin-панель;
- просмотр пользователей;
- модерация мастеров;
- проверка документов;
- управление категориями;
- просмотр заявок и откликов;
- обработка жалоб;
- блокировка пользователей;
- управление балансами и транзакциями;
- просмотр статистики.

## Статусы Заказа

- `draft` - черновик;
- `published` - опубликован;
- `collecting_responses` - собирает отклики;
- `master_selected` - мастер выбран;
- `in_progress` - работа выполняется;
- `completed` - завершен;
- `cancelled` - отменен;
- `disputed` - спор.

## Статусы Мастера

- `new` - новый профиль;
- `profile_incomplete` - профиль не заполнен;
- `pending_verification` - ожидает проверки;
- `verified` - проверен;
- `rejected` - верификация отклонена;
- `blocked` - заблокирован.

### Админ

Admin-панель можно делать после мобильного MVP. На первом этапе достаточно backend API и минимальной web-панели.

## MVP Границы

В первую рабочую версию входят:

- авторизация по телефону с тестовым SMS-режимом;
- роли заказчик/мастер;
- категории;
- создание заявки;
- список заявок;
- отклики мастеров;
- чат;
- профиль мастера;
- кошелек мастера;
- ручная верификация мастера;
- базовая админка для проверки мастеров.

Не входит в первый MVP:

- сложные платежи;
- карта в реальном времени;
- автоматический KYC;
- умное распределение заказов;
- полноценная система споров;
- многоязычность, кроме базовой подготовки интерфейса.

## Нефункциональные Требования

- API должен быть готов к мобильным клиентам;
- все важные действия должны логироваться;
- данные пользователей и токены должны храниться безопасно;
- фото и документы не должны храниться прямо в базе;
- backend должен поддерживать PostgreSQL;
- Flutter-приложение должно работать на Android первым приоритетом;
- дизайн должен оставаться близким к текущему USTO-прототипу.
