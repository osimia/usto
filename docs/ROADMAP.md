# USTO Roadmap

## Фаза 0: Подготовка

Цель: зафиксировать продукт, API и архитектуру перед переписыванием demo-кода.

Задачи:

- утвердить роли и список функций;
- утвердить MVP;
- утвердить API;
- утвердить схему PostgreSQL;
- выбрать state management для Flutter;
- выбрать способ деплоя backend.

Результат:

- готовые документы в `docs/`;
- понятный план разработки.

## Фаза 1: Backend Foundation

Цель: превратить Go-прототип в нормальный API.

Задачи:

- разделить `main.go` на пакеты;
- добавить конфиг через env;
- подключить PostgreSQL;
- добавить SQL-миграции;
- добавить единый формат ошибок;
- добавить middleware логирования;
- добавить JWT auth;
- добавить refresh token;
- добавить базовые CRUD для категорий, заявок и профиля.

Результат:

- backend запускается с PostgreSQL;
- API можно использовать из Flutter.

## Фаза 2: Flutter Foundation

Цель: создать мобильное приложение и подключить базовую авторизацию.

Задачи:

- создать Flutter-проект;
- настроить темы и дизайн-систему;
- добавить роутинг;
- добавить API-клиент;
- добавить secure storage;
- реализовать Splash, Onboarding, Auth;
- подключить `/auth/request-code`, `/auth/verify-code`, `/me`.

Результат:

- пользователь входит в приложение;
- роль сохраняется;
- работает базовая навигация.

## Фаза 3: Customer MVP

Цель: заказчик может создать заявку и выбрать мастера.

Задачи:

- экран категорий;
- форма создания заявки;
- загрузка фото заявки;
- список моих заявок;
- детали заявки;
- список откликов;
- профиль мастера;
- выбор мастера;
- завершение заказа;
- отзыв.

Результат:

- полный путь заказчика от заявки до отзыва.

## Фаза 4: Master MVP

Цель: мастер может найти заявку и отправить отклик.

Задачи:

- профиль мастера;
- выбор услуг;
- портфолио;
- лента заявок;
- фильтры;
- детали заявки;
- форма отклика;
- список моих откликов;
- активные заказы;
- кошелек.

Результат:

- полный путь мастера от регистрации до отклика и выполнения заказа.

## Фаза 5: Chat and Notifications

Цель: пользователи могут быстро договариваться.

Задачи:

- chats API;
- messages API;
- экран списка чатов;
- экран сообщений;
- read state;
- push-уведомления через Firebase Cloud Messaging;
- in-app уведомления.

Результат:

- заказчик и мастер общаются внутри приложения.

## Фаза 6: Verification and Admin

Цель: контролировать качество мастеров и платформы.

Задачи:

- загрузка документов;
- статусы проверки;
- admin API;
- минимальная admin-панель;
- approve/reject мастера;
- блокировка пользователя;
- просмотр жалоб.

Результат:

- администратор может управлять платформой.

## Фаза 7: Payments and Production

Цель: подготовить проект к реальным пользователям.

Задачи:

- подключить платежный провайдер или ручной top-up;
- аудит транзакций;
- rate limiting;
- мониторинг;
- логи;
- backups;
- CI/CD;
- staging и production окружения;
- privacy policy и terms.

Результат:

- проект готов к закрытому запуску.

## Приоритет Разработки

1. Backend auth + PostgreSQL.
2. Flutter auth + навигация.
3. Заявки заказчика.
4. Лента и отклики мастера.
5. Чат.
6. Верификация.
7. Админка.
8. Платежи.

## Рекомендуемый Стек

Backend:

- Go;
- PostgreSQL;
- goose migrations;
- JWT;
- structured logging;
- S3-compatible storage для файлов.

Mobile:

- Flutter;
- Riverpod;
- GoRouter;
- Dio;
- Flutter Secure Storage;
- Firebase Cloud Messaging.

Admin:

- React или простой Go web admin;
- можно начать с минимальной панели без сложного UI.
